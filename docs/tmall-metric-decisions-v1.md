# 天猫指标口径决策 V1

创建时间：2026-06-19

适用范围：V0.3B-1 天猫四源标准数据底座。

本文件锁定四源管道的正式指标口径。旧版 `runAnalysis` 暂不替换，页面、千问和看板功能不在本阶段开发。

## 1. 经营指标

| 指标 | 正式口径 | 分母为 0 |
|---|---|---|
| GMV | 生意参谋 `支付金额` | 不适用 |
| 核心退款金额 | 生意参谋 `成功退款金额` | 不适用 |
| GSV | 支付金额 - 成功退款金额 | 不适用 |
| 退款率 | 成功退款金额 / 支付金额 | 返回 `null` |
| 支付转化率 | 支付买家数 / 商品访客数 | 返回 `null` |
| 客单价 | 支付金额 / 支付买家数 | 返回 `null` |
| 日均 GMV | 统计周期 GMV / 统计天数 | 返回 `null` |

规则：

1. 所有分母为 0、空值或无效值时返回 `null`。
2. 不允许返回 `Infinity`、`-Infinity` 或 `NaN`。
3. 比率类指标在商品、系列、店铺汇总时必须用汇总后的分子和分母重新计算，不能直接平均。

## 2. 推广指标

| 指标 | 正式口径 |
|---|---|
| 商品维度推广指标 | 使用商品报表 |
| 计划维度推广指标 | 使用计划报表 |
| 店铺推广总量 | 优先使用计划报表 |
| 推广成交金额 V1 | 使用 `总成交金额`，不单独叠加预售成交金额 |
| 投入产出比 | 总成交金额 / 花费 |
| 推广费比 | 推广花费 / GMV |
| 去退推广费比 | 推广花费 / GSV |
| 直接成交占比 | 直接成交金额 / 总成交金额 |
| 间接成交占比 | 间接成交金额 / 总成交金额 |

硬规则：

1. 商品报表和计划报表不得直接相加。
2. 商品报表只用于商品分摊和商品下钻。
3. 计划报表优先用于计划看板和店铺推广总量。
4. 比率类指标汇总时必须重算，不能直接平均。
5. V0.3A 审计报告中 `ad_cost_ratio` 曾写成 `花费 / 总成交金额`，该公式不是本项目正式口径。
6. 正式口径为 `ad_cost_ratio = ad_spend / gmv`。
7. 正式口径为 `ad_cost_ratio_after_refund = ad_spend / gsv`。

## 3. 售后指标

| 指标/场景 | 正式口径 |
|---|---|
| 申请数量 | 按 `退款申请时间` 统计 |
| 申请金额 | 按 `退款申请时间` 统计 `退款总额` |
| 成功数量 | 按 `退款完结时间` 统计 |
| 成功金额 | 按 `退款完结时间` 分别保留 `退款总额`、`退给买家金额`、`退给平台金额` |
| 退款销售归因 | 按 `订单付款时间` 统计 |
| 退款原因 | 优先使用 `买家退款原因` |
| 退款原因标签 | 当前样本为空，不作为 V1 来源 |
| 超时未处理 | `analysisTimestamp > 超时时间` 且退款状态属于待处理集合 |
| GSV | 仍以生意参谋 `成功退款金额` 计算，不直接用售后表替换 |

规则：

1. `退款总额`、`退给买家金额`、`退给平台金额` 分别保留，不混成一个字段。
2. 售后表用于原因、状态、处理效率和退款明细汇总分析。
3. 售后表不得输出原始行。
4. 售后敏感字段不得进入 localStorage。
5. 超时判断使用本次分析的 `analysisTimestamp`，不使用页面实时刷新时间，保证结果可复现。

## 4. 售后状态集合

成功状态：

- `退款成功`

待处理状态：

- `买家已经申请退款，等待卖家同意`
- `卖家已经同意退款，等待买家退货`
- `买家已经退货，等待卖家确认收货`

未知状态：

1. 未来出现但未识别的状态不得自动视为成功。
2. 未来出现但未识别的状态不得自动视为待处理。
3. 未知状态归入 `unknownStatus`。
4. 未知状态必须进入数据质量警告。

## 5. 关联指标命名

推广商品关联成功率：

```text
advertisedProductJoinRate = 成功关联的推广主体数 / 商品报表主体数
```

当前样本预期：`9 / 9 = 100%`

全店商品推广覆盖率：

```text
storePromotionCoverage = 有推广数据的商品数 / 生意参谋商品数
```

当前样本预期：`9 / 19 = 47.37%`

规则：

1. 未出现在商品报表中的经营商品不视为关联失败。
2. 未出现在商品报表中的经营商品 `hasAdData = false`。
3. 未出现在商品报表中的经营商品仍保留经营数据。
4. 后续页面显示为“暂无推广数据”。
5. 不得删除这些商品。
