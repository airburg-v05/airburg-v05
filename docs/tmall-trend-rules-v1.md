# 天猫趋势数据规则 V1

本文档定义 V0.4B-1 阶段的趋势数据口径。当前阶段只建立数据模型与 ViewModel，不开发趋势图、不接入图表库、不做目标设置、不做 MTD/DLY、不做预测、不接 AI、不接后端和数据库。

## 数据来源

1. 经营趋势来自 `productDailyFacts`。
   - 店铺经营趋势：按日期汇总全部商品。
   - 商品经营趋势：按商品 ID 过滤后按日期汇总。
   - 系列经营趋势：按系列中的商品 ID 过滤后按日期汇总。

2. 店铺推广趋势来自 `adPlanDailyFacts`。
   - 店铺级推广花费、推广成交金额、推广 ROI、点击量、点击单价、点击率使用计划报表按日期汇总。
   - 不允许使用 7 日计划报表总量伪造成经营趋势。

3. 商品推广趋势来自 `adProductDailyFacts`。
   - 商品级推广花费、点击量、点击单价、推广成交金额、推广 ROI 按商品 ID 和日期汇总。

4. 系列推广趋势来自系列商品 ID 与 `adProductDailyFacts`。
   - 先使用系列商品 ID 过滤商品推广事实表，再按日期汇总。
   - 系列推广费比使用同一天的系列经营 GMV 作为分母。
   - 系列去退推广费比使用同一天的系列经营 GSV 作为分母。

5. 售后趋势来自 `afterSalesAggregates`。
   - `applyDate` 使用申请时间口径。
   - `successDate` 使用退款完结时间口径。
   - `paymentDate` 使用订单付款时间口径。
   - 售后趋势只使用聚合后的安全结果，不读取或输出售后原始订单、地址、电话、备注等敏感明细。

## 指标计算规则

1. 加总指标按日期求和。
   - GMV、GSV、成功退款金额、访客数、支付买家数、推广花费、推广成交金额、点击量、退款申请数等，均按同一天的明细求和。

2. 比率指标必须按每日分子和分母重新计算。
   - 支付转化率 = 当日支付买家数合计 / 当日商品访客数合计。
   - 客单价 = 当日 GMV 合计 / 当日支付买家数合计。
   - 退款率 = 当日成功退款金额合计 / 当日 GMV 合计。
   - 推广 ROI = 当日推广成交金额合计 / 当日推广花费合计。
   - 推广点击单价 = 当日推广花费合计 / 当日点击量合计。
   - 推广点击率 = 当日点击量合计 / 当日展现量合计。
   - 推广费比 = 当日推广花费合计 / 当日 GMV 合计。
   - 去退推广费比 = 当日推广花费合计 / 当日 GSV 合计。

3. 分母为 0 时返回 `null`。
   - 不允许返回 `NaN`。
   - 不允许返回 `Infinity`。
   - 不允许返回 `undefined`。

## 趋势可用性规则

1. 只有 1 个日期点时，标记 `insufficientDataForTrend`。
   - 单点数据可以展示为当日值，但不建议解释为趋势。

2. 没有对应来源数据时，返回空趋势并标记 `missingDataSource`。
   - 示例：没有商品推广报表时，商品推广趋势返回空数组。

3. 不同来源的日期范围独立判断。
   - 生意参谋商品表只有 1 天时，经营趋势为单点。
   - 计划推广报表有 7 天时，店铺推广趋势可以形成 7 个日期点。
   - 不得把推广 7 天数据补成经营 7 天数据。

4. ViewModel 必须是纯函数。
   - 不读取 `localStorage`。
   - 不访问 `window`。
   - 不使用 React 状态。
   - 不修改传入的分析结果对象。

## 当前不包含

1. 趋势图 UI。
2. 图表库。
3. 目标设置。
4. MTD/DLY。
5. 趋势预测。
6. AI 分析。
7. 后端接口。
8. 数据库存储。
9. 自动平台 API。
