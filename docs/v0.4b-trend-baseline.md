# V0.4B 多日趋势基础版稳定基线

## 1. 版本定位

V0.4B 是天猫四源本地分析平台的“多日趋势基础版”。

当前版本只基于已经上传并安全聚合的数据做趋势展示，不接后端、不接数据库、不接 AI、不接自动平台 API。趋势层服务于运营判断：当前数据是否具备趋势观察条件、趋势来自哪个数据源、详细趋势应该进入哪个看板继续查看。

当前仍不包含：

1. 目标设置。
2. MTD/DLY。
3. 趋势预测。
4. AI 分析。
5. 数据库。
6. 后端接口。
7. 自动平台 API。
8. 正式图表库。
9. 数据导出。

## 2. 已完成能力

V0.4B 已完成以下能力：

1. 趋势数据规则文档：`docs/tmall-trend-rules-v1.md`。
2. 趋势 ViewModel 基础层：`lib/tmall/view-models/trends.ts`。
3. 店铺看板趋势展示基础版。
4. 宝贝看板趋势展示基础版。
5. 系列看板趋势展示基础版。
6. 首页趋势摘要轻量版。
7. 私密验证脚本。

1. 首页趋势摘要。
2. 店铺趋势分析。
3. 宝贝趋势分析。
4. 系列趋势分析。

## 3. 趋势数据来源

趋势层只读取四源分析后的安全聚合结果：

1. 经营趋势来自 `productDailyFacts`。
2. 店铺推广趋势来自 `adPlanDailyFacts`。
3. 宝贝推广趋势来自 `adProductDailyFacts`。
4. 系列推广趋势来自系列商品 ID 对应的 `adProductDailyFacts`。
5. 售后趋势来自 `afterSalesAggregates`。
6. 不使用售后原始明细。
7. 不把计划推广 7 日数据补成经营趋势。
8. 不把商品推广和计划推广相加。

不同数据源的日期范围独立判断。经营数据、计划推广数据、商品推广数据、售后数据可以拥有不同的日期点数量，趋势层不得强行补齐或混用。

## 4. 当前样本数据表现

基于当前 `private-samples/tmall` 四份真实样表，V0.4B 当前样本状态为：

1. 生意参谋商品表经营数据：1 个日期点。
2. 商品推广数据：1 个日期点。
3. 计划推广数据：7 个日期点。
4. 售后申请数据：1 个日期点。
5. 首页经营趋势状态：`single_point`。
6. 首页计划推广趋势状态：`ready`。

说明：

当前经营趋势只能展示当日值，不解释为趋势；计划推广数据可以观察 7 日推广趋势。系统不会用计划推广 7 日数据补齐经营趋势。

## 7. 趋势状态规则

`pointCount = 0`：

1. 暂无趋势数据。
2. `missingDataSource = true`。

`pointCount = 1`：

1. 当前指标只有 1 个日期点。
2. `insufficientDataForTrend = true`。
3. 只展示当日值，不解释为趋势。

`pointCount >= 2`：

1. 可观察趋势。

通用安全规则：

1. `null` 不当作 0。
2. 不返回 `NaN`。
3. 不返回 `Infinity`。
4. 不返回 `undefined`。
5. 空值统一显示 `--`。

## 10. 当前安全边界

趋势层不得输出：

1. 售后原始明细。
2. 订单号。
3. 退款编号。
4. 手机号。
5. 地址。
6. 物流信息。
7. 买家退款说明。
8. 商家备注。
9. 原始 `rows`。
10. `previewRows`。
11. 文件名。
12. `localStorage` key。
13. API key。

趋势层只能面向页面输出聚合后的业务指标、日期点数量、可观察性状态和安全提示。

## 11. 冻结规则

V0.4B 后续小修不得改动：

1. 四源解析规则。
2. GB18030 解码。
3. 商品、计划、售后关联规则。
4. 正式指标口径。
5. `airburg_tmall_analysis_v2` 存储结构。
6. `airburg_tmall_series_groups_v1` 存储结构。
7. 趋势基础口径。
8. 趋势状态规则。
9. 无推广数据不得显示为 0 的规则。
10. 单点数据不得解释为趋势的规则。

如后续发现展示问题，只允许做不改变趋势口径和存储结构的小范围 UI 修正。
