# 统一字段字典

## 1. 使用说明

本字段字典用于把天猫、京东、抖音等平台导出的不同表头，统一映射成平台内部可计算的标准字段。

当前字典是第一阶段草案。字段映射必须以后续真实上传表格为证据持续校准，不能仅凭字段名称做最终经营判断。

字段识别原则：

- 能明确匹配的字段，自动映射。
- 多个候选字段含义相近时，提示用户确认。
- 字段缺失时，标记为缺失，不用 0 代替未知。
- 字段含义不确定时，标记为待确认。
- 平台导出字段口径发生变化时，更新本字典。

## 2. 数据类型约定

| 数据类型 | 含义 | 示例 |
|---|---|---|
| string | 文本 | 商品名称、SKU名称 |
| enum | 枚举 | tmall、jd、douyin |
| date | 日期 | 2026-06-15 |
| integer | 整数 | 访客数、订单数 |
| decimal | 小数或金额 | 12999.50 |
| percentage | 百分比 | 3.25% |

## 4. 衍生指标计算口径

| 指标 | 计算公式 | 前置字段 | 注意事项 |
|---|---|---|---|
| 支付转化率 | `paid_buyers / visitors` | `paid_buyers`、`visitors` | 当 `visitors` 为 0 或缺失时不可计算。 |
| 客单价 | `sales_amount / paid_buyers` | `sales_amount`、`paid_buyers` | 当 `paid_buyers` 为 0 或缺失时不可计算。 |
| 浏览深度 | `page_views / visitors` | `page_views`、`visitors` | 第一阶段可预留，不作为核心验收指标。 |
| 退款率 | `refund_orders / paid_orders` | `refund_orders`、`paid_orders` | 订单口径不一致时不能直接计算。 |
| 退款金额占比 | `refund_amount / sales_amount` | `refund_amount`、`sales_amount` | 销售额为 0 或缺失时不可计算。 |

## 5. 字段质量标记

每个字段映射后建议保留质量标记：

| 标记 | 含义 |
|---|---|
| matched | 字段明确匹配 |
| calculated | 字段由公式计算 |
| missing | 原始表格缺失 |
| ambiguous | 字段含义可能有多个解释 |
| invalid | 字段存在无法解析的数据 |
| manual_confirmed | 已由用户人工确认 |

## 6. 更新规则

后续每新增一种平台导出表或一种新表头，都要更新：

1. 对应平台字段候选名。
2. 字段清洗规则。
3. 是否能用于核心指标计算。
4. 是否需要用户确认。
5. 是否影响历史数据口径。
