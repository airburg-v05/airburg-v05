# V0.4C 目标设置与看板联动基础版稳定基线

## 1. 版本定位

V0.4C：目标设置与看板联动基础版。

当前状态：PASS / 待用户最终确认后冻结。

V0.4C 基于天猫四源本地分析结果和本地目标配置，实现店铺、宝贝、系列三类目标的创建、编辑、暂停、启用、删除，以及首页、店铺看板、宝贝看板、系列看板的目标完成率摘要展示。

当前版本只基于已经上传并安全聚合的数据做目标完成率计算，不接后端、不接数据库、不接 AI、不接自动平台 API。

## 2. 已完成模块

V0.4C 已完成以下模块：

1. 目标 storage。
2. 目标计算底层。
3. 店铺目标管理。
4. 宝贝目标管理。
5. 系列目标管理。
6. 首页目标摘要。
7. 店铺看板目标摘要。
8. 宝贝看板目标摘要。
9. 系列看板目标摘要。

## 3. Storage Key

当前目标体系相关本地存储 key：

1. `airburg_tmall_targets_v1`
2. `airburg_tmall_analysis_v2`
3. `airburg_tmall_series_groups_v1`

`airburg_tmall_targets_v1` 只保存目标定义，不保存四源分析结果、商品名称、系列商品列表、原始表格行或售后敏感明细。

## 4. 支持范围

支持目标 scope：

1. `store`
2. `product`
3. `series`

支持目标 period：

1. `daily`
2. `monthly`

支持指标：

1. GMV。
2. GSV。
3. 商品访客数。
4. 支付买家数。
5. 支付转化率。
6. 客单价。
7. 退款率。
8. 推广花费。
9. 推广 ROI。
10. 推广费比。
11. 去退推广费比。

支持指标方向：

1. `higher_is_better`
2. `lower_is_better`

## 5. 完成率规则

通用目标完成率规则：

1. 越高越好：`actual / target`。
2. 越低越好：`target / actual`。
3. `actual = null`：状态为 `missing_actual`。
4. `target` 非法：状态为 `invalid_target`。
5. `paused`：状态为 `paused`。

状态含义：

1. `achieved`：已达到或优于目标。
2. `in_progress`：接近目标。
3. `at_risk`：低于目标安全区间，需要关注。
4. `missing_actual`：缺少当前实际值或关联数据。
5. `invalid_target`：目标值非法。
6. `paused`：目标已暂停。

月度目标只基于已上传日期计算，当前不做 MTD/DLY、预测或自然月补齐。

## 6. 页面职责

`/targets`：

1. 创建目标。
2. 编辑目标。
3. 暂停目标。
4. 启用目标。
5. 删除目标。
6. 管理 store / product / series 三类目标。

`/home`：

1. 展示全局目标摘要。
2. 统计 store / product / series 三类目标整体完成情况。
3. 不承担目标编辑职责。

`/store-board`：

1. 只展示店铺目标摘要。
2. 只统计 `scope === "store"` 的目标。
3. 目标创建和编辑仍进入 `/targets`。

`/product-board`：

1. 只展示当前商品目标摘要。
2. 只统计 `scope === "product"` 且 `productId` 等于当前选中商品 ID 的目标。
3. 切换商品后目标摘要同步切换。
4. 目标创建和编辑仍进入 `/targets`。

`/series-board`：

1. 只展示当前系列目标摘要。
2. 只统计 `scope === "series"` 且 `seriesId` 等于当前选中系列 ID 的目标。
3. 切换或编辑系列后目标摘要使用当前系列商品 ID 重新计算。
4. 目标创建和编辑仍进入 `/targets`。

## 7. 数据来源

目标实际值只来自四源安全聚合结果：

1. 经营目标来自 `productDailyFacts`。
2. 店铺推广目标来自 `adPlanDailyFacts`。
3. 宝贝推广目标来自 `adProductDailyFacts`。
4. 系列推广目标来自系列商品 ID 对应的 `adProductDailyFacts`。
5. 售后相关指标只使用安全聚合后的退款金额，不使用售后原始明细。

关键边界：

1. 不把计划推广数据补成宝贝推广目标。
2. 不把计划推广数据补成系列推广目标。
3. 不把商品推广和计划推广相加。
4. 无推广商品和无推广系列的推广实际值为 `null`，页面显示 `--`，不得显示为 0。

## 8. 安全边界

目标体系不得保存或输出：

1. 商品名称作为目标主键。
2. 系列商品列表到目标 storage。
3. 原始 `rows`。
4. `previewRows`。
5. 售后敏感字段。
6. 售后敏感字段值。
7. 文件名。
8. `localStorage` key 到业务 ViewModel。
9. 订单号。
10. 退款编号。
11. 手机号。
12. 地址。
13. 物流信息。
14. 买家退款说明。
15. 商家备注。

目标 storage 只保存目标定义，包括 scope、period、metric、targetValue、direction、status 和对应 scope 的必要 ID。

## 9. 当前不做

V0.4C 当前不包含：

1. 目标趋势联动。
2. MTD / DLY。
3. 预测。
4. AI 诊断。
5. 导出报告。
6. 数据库。
7. 后端 API。
8. 自动采集。
9. 多店铺目标体系。
10. 用户权限。

## 10. 损坏状态规则

目标 storage 损坏：

1. 页面显示安全提示。
2. 不自动删除损坏数据。
3. 清除损坏目标数据只在 `/targets` 处理。
4. 首页、店铺看板、宝贝看板、系列看板不得崩溃。

系列 storage 损坏：

1. 系列看板显示安全提示。
2. 不自动删除损坏数据。
3. 不影响 store / product 目标摘要计算。
4. 缺失或找不到 `seriesId` 时，系列目标实际值为 `null`，状态为 `missing_actual`。

## 12. 冻结规则

V0.4C 后续小修不得随意修改：

1. 目标 storage 结构。
2. 目标完成率口径。
3. 无推广数据 `null` 规则。
4. 单点数据不解释为趋势规则。
5. 四源底层解析规则。
6. GB18030 解码。
7. 商品、计划、售后关联规则。
8. 正式指标口径。
9. `airburg_tmall_analysis_v2` 存储结构。
10. `airburg_tmall_series_groups_v1` 存储结构。
11. `airburg_tmall_targets_v1` 存储结构。

如果后续确实需要修改以上内容，必须先单独立项、补充口径说明和回归脚本，再执行迁移或兼容策略。

V0.4C 冻结后，建议进入 V0.4D 或后续阶段时优先考虑：

3. 目标报表导出。
