# V0.4E 数据接入闭环冻结说明

## 1. 阶段名称

V0.4E：数据接入闭环与安全查看基础版。

本阶段目标是让运营从进入系统、查看全局数据状态、补齐上传缺口、复核安全聚合数据，到进入首页和各看板形成完整闭环。

## 2. 完成范围

V0.4E 已完成：

1. 首页经营工作台总览。
2. 首页日期联动和模块导航。
3. 全局数据状态提醒。
4. 上传页数据质量中心。
5. 原始数据安全查看中心。
6. 数据接入缺口引导。
7. warning 脱敏展示。
8. empty / corrupted / valid 状态安全处理。
9. 390px 移动端基础可用性回归。

## 3. 已冻结模块

当前冻结模块包括：

1. `/home` 经营工作台首页。
2. `/upload` 数据上传与数据质量中心。
3. `/raw-data` 原始数据安全查看中心。
4. `/store-board` 店铺看板基础版。
5. `/series-board` 系列看板基础版。
6. `/product-board` 宝贝看板 V0.4D 基础版。
7. `/targets` 目标管理基础版。
8. `/login` 登录占位页。

## 4. AI 和依赖边界

本阶段明确未接入：

1. AI 分析。
2. 千问、百炼、DashScope、OpenAI 等外部模型服务。
3. 后端接口。
4. 数据库。
5. 自动平台 API。
6. 新增业务依赖。

本阶段未新增 `package.json` 依赖。

## 5. 四源底层边界

本阶段未修改：

1. 四源底层解析逻辑。
2. `run-tmall-four-source-analysis`。
3. `table-parser`。
4. GB18030 解码规则。
5. 商品、计划、售后关联规则。
6. V0.4C 目标口径。
7. V0.4B 趋势口径。
8. storage 结构。

## 6. 隐私边界

页面只展示安全聚合结果，不展示售后原始明细。

前台不得展示：

1. 售后原始 rows。
2. 原始 previewRows。
3. 文件内容原文。
4. JSON 大对象原文。
5. 技术报错堆栈。
6. 售后敏感字段名。
7. 售后敏感源值。

上传页和原始数据页的 warning 只展示脱敏后的安全提示，最多展示 5 条。

## 8. 状态矩阵

V0.4E 冻结时覆盖：

1. 无分析结果：引导去上传，页面安全 empty。
2. 分析结果损坏：提示重新上传，不渲染原始明细。
3. valid 分析结果：首页、上传页、raw-data 和各看板可生成。
4. 四源缺口：全局提醒和上传页补齐清单提示缺失来源。
5. 数据质量 warning：首页、全局提醒、上传页和 raw-data 只展示安全提示。
6. target storage corrupted：全局提醒指向目标管理，页面不崩溃。
7. series storage corrupted：全局提醒指向系列看板，其他页面不崩溃。

## 10. 已知非阻断问题

1. `favicon.ico` 可能出现 404，属于非业务资源问题，不作为 V0.4E 阻断。
2. 当前仍为本地浏览器数据分析，不包含登录鉴权、后端数据库和多用户同步。

## 11. 下一阶段建议

V0.4F 建议进入“经营目标与动作闭环”。

建议拆分方向：

1. 目标管理可用性增强。
2. 目标模板。
3. 目标完成趋势。
4. 手动运营动作记录。
5. 非 AI 规则化复盘。

V0.4F 不应修改 V0.4E 数据接入闭环的隐私边界和四源底层解析规则。
