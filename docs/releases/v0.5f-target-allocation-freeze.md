状态：PASS，immutable completion record 已注册，V0.5F 已作为冻结基线。

## 阶段范围

V0.5F 冻结公司总目标、店铺目标、系列目标和商品目标的层级管理能力。目标体系基于显式 `parentTargetId`，支持 `company → store → series → product` 的手动分配，不根据 scope、名称、指标或历史表现自动猜测父子关系。

本阶段已完成：

1. V0.5F-0：目标层级合同与 storage readiness。
2. V0.5F-1-R1：V2-first 目标管理中心和目标抽屉。
3. V0.5F-2：父子目标手动分配工作流。
4. V0.5F-3：首页、店铺、系列和宝贝看板紧凑目标摘要接入。
5. V0.5F-4：目标运行时、视觉、冲突和截图证据收口。
6. V0.5F-5-R1：最终回归和阶段冻结。

旧任务 `V0.5F_5_FINAL_REGRESSION_AND_STAGE_FREEZE` 保留为 BLOCKED 历史证据，不改写为 PASS。

## 已冻结能力

1. `/targets` 为 V2-first 目标管理中心。
2. 支持 `company`、`store`、`series`、`product` 四种 scope。
3. 新建、编辑、暂停和重新启用目标均通过目标抽屉完成。
4. 目标保存使用 prepare / readback / activate 原子流程。
5. 使用 `expectedCurrentDatasetId` 防止并发 active pointer 冲突覆盖。
6. 暂停目标不硬删除，重新启用保持原 targetId 和 parentTargetId。
7. legacy 旧目标继续保留，非默认店铺不写 legacy target key。
8. 四级看板只展示当前上下文匹配的紧凑目标摘要。

## 目标层级规则

1. `company` 目标可作为 `store` 目标直接父级。
2. `store` 目标可作为同 platformCode + storeId 的 `series` 目标直接父级。
3. `series` 目标可作为同 platformCode + storeId + seriesId 的 `product` 目标直接父级。
4. 不允许跳级分配。
5. 不允许跨店铺分配。
6. 子目标必须继承并锁定父目标的 metricKey、periodType、periodValue 和 direction。
7. 用户手动填写每个子目标 targetValue。
8. 不自动平均分配、不按历史销售比例分配、不按店铺贡献分配、不接 AI 分配。
9. 对转化率、ROI 等不可加总指标不做层级分配。

## 分配状态

父目标分配摘要展示：

1. 父目标值。
2. active 已分配。
3. paused 已分配。
4. 剩余值。
5. 超额值。
6. 子目标数量。
7. allocationStatus。

状态固定为：

1. `under_allocated`：active 子目标合计小于父目标。
2. `fully_allocated`：active 子目标合计与父目标一致。
3. `over_allocated`：active 子目标合计超过父目标。

超额分配允许保存，但必须展示风险提示。paused 子目标保留关系，但不计入 active 已分配。

## 页面职责

1. `/targets`：创建、编辑、暂停、重新启用目标，查看父子分配。
2. `/home`：只展示 company 目标摘要。
3. `/store-board`：只展示当前 platformCode + storeId 的 store 目标摘要。
4. `/series-board`：只展示当前 platformCode + storeId + seriesId 的 series 目标摘要。
5. `/product-board`：只展示当前 platformCode + storeId + productId 的 product 目标摘要。

各看板只展示紧凑目标摘要，不恢复长目标诊断卡墙，不显示目标编辑表单。

英文页面名对照：Home、Store Board、Series Board、Product Board。

## 周期和完成率边界

1. 保留 V0.4C 目标完成率口径。
2. 日目标仅用于日视图。
3. 月目标仅用于月视图。
4. 周和自定义范围没有明确目标时显示无目标，不自动折算。
5. 不修改首页、店铺、系列、宝贝看板的实际指标、趋势、排序或筛选规则。
6. 不把商品或系列目标自动汇总为公司目标。

## 隐私和安全

目标体系不得保存或展示：

1. 售后原始 rows。
2. previewRows。
3. 文件名历史。
4. 订单号、退款编号、交易号。
5. 联系方式、地址、物流信息。
6. 买家退款说明、商家备注。
7. 操作人、子账号。
8. 技术错误堆栈。

目标记录只允许保存目标身份、scope、ownership、period、metric、direction、targetValue、parentTargetId、status、createdAt 和 updatedAt 等安全字段。

6. legacy 旧目标保留规则。
