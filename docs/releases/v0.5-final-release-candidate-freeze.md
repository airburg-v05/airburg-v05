## 已冻结能力

1. 四源批量导入、自动识别、导入历史和数据质量中心。
2. V2 多平台、多店铺数据归属和本地 IndexedDB 原子激活。
3. 首页跨店铺经营命令中心。
4. 多店铺 Store Board。
5. 店铺隔离的系列管理和重点商品管理。
6. 用户定义 Series Board 和 Tracked Product Board。
7. company → store → series → product 四级目标和手动分配。
8. Raw Data 安全查看中心。
9. legacy fallback 与旧数据保留边界。
10. 全站视觉、导航、可访问性和移动端收口。

## 数据和指标口径

1. 保持四源解析、GB18030、商品/计划/售后关联规则不变。
2. 保持 GMV、GSV、访客、买家、转化率、推广花费和 ROI 既有冻结口径。
3. 推广不重复计算，商品推广不使用计划推广补齐。
4. 缺失值显示为 --，不按 0 计算；真实 0 保留为 0。
5. 转化率使用汇总支付买家 / 汇总访客，不平均商品或店铺转化率。
6. 目标完成率沿用 V0.4C 冻结规则。
7. 周和自定义周期无明确目标时不折算。

## 多店铺隔离

1. 所有 V2 数据必须绑定 platformCode + storeId。
2. 相同 productId 跨店铺不得合并。
3. 同名 series 跨店铺不得混合。
4. 非默认店铺不得写 legacy target key 或 legacy series key。
5. 默认店铺 legacy fallback 不自动迁移、不写 V2。

## 目标层级

1. 支持 company、store、series、product 四种 scope。
2. 父子关系使用 parentTargetId 显式绑定。
3. 不根据 scope、名称、指标或周期自动猜父目标。
4. 支持 under_allocated、fully_allocated、over_allocated。
5. paused 目标保留关系，不计入 active 分配。

## 隐私边界

不得在 DOM、console、URL、IndexedDB 输出、截图或测试输出中展示：

1. rawRows、previewRows、原始文件内容。
2. 文件名历史和 warning 原文。
3. 订单、退款、交易标识。
4. 电话、地址、物流信息。
5. 买家说明、商家备注、操作人或子账号。
6. 技术错误堆栈。

售后只展示安全聚合，不展示敏感明细。
