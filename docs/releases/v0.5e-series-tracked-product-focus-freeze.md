# V0.5E Series And Tracked Product Focus Freeze

冻结时间：2026-06-23T01:13:29+08:00

阶段名称：V0.5E：系列与重点商品自定义管理基础版

状态：PASS，待 immutable completion record 注册后生效。

## 1. 阶段定位

V0.5E 在 V0.5A-D 的多平台、多店铺数据地基、数据中心、首页命令中心和多店铺 Store Board 之上，完成运营主动定义焦点对象的基础能力：

1. 按 `platformCode + storeId` 隔离的系列管理。
2. 按 `platformCode + storeId` 隔离的重点商品管理。
3. 只展示用户启用系列的 Series Board。
4. 只展示用户启用重点商品的 Product Board。

本阶段不接 AI，不接后端，不新增依赖，不修改四源解析、迁移、V2 persistence、指标口径、目标口径和趋势口径。

## 2. 系列管理定位

系列管理入口为：

`/series-board/manage?platform=<platformCode>&storeId=<storeId>`

系列管理只负责维护当前店铺的用户自定义系列，不做看板指标计算，不做目标编辑，不自动生成系列。

系列身份由以下字段共同确定：

1. `platformCode`
2. `storeId`
3. `seriesId`

系列记录只允许保存安全字段：

1. ID。
2. platformCode。
3. storeId。
4. 名称。
5. productIds。
6. status。
7. createdAt。
8. updatedAt。

## 3. 系列创建、编辑、停用和重新启用

冻结规则：

1. 系列名称必填，trim 后长度 2-40。
2. 同平台、同店铺名称归一化后不可重复。
3. 不同店铺允许同名系列，即跨店同名系列允许存在且互不混合。
4. active 和 inactive 系列都参与重复名称检查。
5. 系列商品只能来自当前店铺商品候选集合。
6. productIds 去重。
7. 系列允许为空。
8. 停用只把 status 改为 `inactive`，不硬删除。
9. 重新启用继续使用原 `seriesId`。
10. createdAt 保持不变，updatedAt 由保存操作更新。

系列管理不得自动按商品名称、类目、销量或推广数据生成系列。

## 4. 重点商品管理定位

重点商品管理入口为：

`/product-board/tracked?platform=<platformCode>&storeId=<storeId>`

重点商品管理只维护当前店铺的用户主动关注商品，不自动把全部商品、销量 TOP、推广高花费商品或售后高金额商品加入重点商品。

重点商品身份由以下字段共同确定：

1. `platformCode`
2. `storeId`
3. `trackedProductId`

商品业务身份使用：

1. `platformCode`
2. `storeId`
3. `productId`

## 5. 重点商品添加、编辑、停用和重新启用

冻结规则：

1. 只能选择当前店铺商品。
2. 同平台、同店铺、同 productId 只能有一条 TrackedProductRecord。
3. 不同店铺允许相同 productId，即跨店同 productId 允许存在且互不混合。
4. displayName 可为空，trim 后最多 60 字符。
5. displayName 不参与商品身份判定。
6. 停用只把 status 改为 `inactive`，不硬删除。
7. 重新启用继续使用原 `trackedProductId`。
8. inactive 后不得创建同 productId 新记录，应重新启用原记录。

## 6. 商品候选和 ad-only 商品

商品候选集合来自当前店铺：

1. businessProductFacts.productId。
2. adProductFacts.productId。

规则：

1. 同 productId 去重。
2. business 数据优先提供名称。
3. 只有推广事实的商品也可选择。
4. ad-only 商品显示“仅推广数据”。
5. 有经营事实显示“有经营数据”。
6. 不读取其他店铺商品。
7. 不把 planId 当成 productId。
8. 不使用文件名推断商品。

## 7. 保存与并发冲突

所有保存必须基于当前 active V2 dataset。

保存流程冻结为：

1. 读取当前 active dataset。
2. 记录 expectedCurrentDatasetId。
3. 对 dataset 做安全副本。
4. 只修改当前店铺 SeriesRecord 或 TrackedProductRecord。
5. 运行完整 V2 validator。
6. 通过现有 prepare/readback/activate 公共能力保存。
7. 激活前检查 expectedCurrentDatasetId。
8. active pointer 冲突时返回 conflict。
9. conflict 不覆盖新 dataset。
10. 保存成功后重新读取最新数据。

禁止页面直接 `indexedDB.open`、直接访问 object store、复制 persistence adapter、写新的 localStorage key、修改 legacy key 或硬删除记录。

## 8. Store Board 入口

Store Board 保留简洁看板职责，只提供轻量入口：

1. 管理系列。
2. 管理重点商品。
3. 进入 Series Board。
4. 进入 Product Board。

入口必须携带 platform/storeId。管理表单不嵌入 Store Board。

## 9. Series Board 定位

`/series-board` 已升级为用户自定义系列看板：

1. 只展示当前 `platformCode + storeId` 下 status = active 的 SeriesRecord。
2. 不自动创建系列。
3. 不自动推导系列。
4. 不展示 inactive 系列作为看板对象。
5. 不把默认店铺系列复制到第二店铺。
6. 同名跨店系列互不混合。

Series Board 展示：

1. 店铺、系列与经营范围控制条。
2. 六个核心指标与系列目标只读摘要。
3. 单一主趋势图。
4. 系列商品组成与轻量数据状态。

默认店铺可 legacy fallback；非默认店铺不得使用 legacy fallback。

## 10. Product Board 定位

`/product-board` 已升级为重点商品优先的宝贝看板：

1. 只展示当前店铺用户启用的 TrackedProductRecord。
2. 不默认铺出全部商品。
3. productId 参数仅做 legacy 兼容跳转。
4. 未设为重点商品的 productId 显示安全提示。
5. 无重点商品时引导到重点商品管理。
6. 同 productId 跨店铺不得混合。

Product Board 展示：

1. 店铺、重点商品与经营范围控制条。
2. 六个核心指标与商品目标只读摘要。
3. 单一主趋势图。
4. 所属系列、推广、售后和轻量数据状态。

## 11. 目标只读边界

V0.5E 只读取已有目标：

1. 系列目标只匹配当前 platform/store/series。
2. 商品目标只匹配当前 platform/store/product。
3. 日目标仅日视图。
4. 月目标仅月视图。
5. 周和自定义不自动折算目标。
6. 不把商品目标加总成系列目标。
7. 不把店铺目标自动分配到系列或商品。
8. 不创建、不编辑、不生成目标。

目标编辑继续留在后续阶段。

## 12. 单一主趋势图

Series Board 和 Product Board 都只保留一个主趋势图，通过 Tab 切换：

1. GMV。
2. GSV。
3. 访客。
4. 支付买家。
5. 转化率。
6. 推广花费。

趋势图不得输出 NaN、Infinity 或 undefined；单日数据不得解释为上涨或下降；缺失日期不补 0。

## 13. 下钻规则

Store Board、Series Board、Product Board 下钻规则：

1. 默认店铺可进入已兼容的默认店铺页面。
2. 非默认店铺必须携带 platform/storeId。
3. Series Board 已支持非默认店铺。
4. Product Board 已支持非默认店铺的重点商品。
5. 未设为重点商品的商品不得伪装成已支持的 Product Board 对象。
6. invalid store / invalid series / invalid tracked product 不自动 fallback 到其他对象。

## 14. legacy 边界

legacy 仅用于默认天猫店铺过渡：

1. 旧系列 key `airburg_tmall_series_groups_v1` 不被写入。
2. 旧分析 key 不被修改。
3. 默认店铺可读取迁移后的 V2 系列和重点商品。
4. 第二店铺不读取 legacy 系列或默认店铺重点商品。
5. 页面不自动迁移。

## 15. 390px 和可访问性

390px 移动端冻结要求：

1. 无整页横向溢出。
2. 系列列表、重点商品列表、抽屉、选择器、Series Board 和 Product Board 可操作。
3. 主趋势图不撑破页面。
4. 长商品名和长店铺名安全截断。
5. 操作按钮可触达。

可访问性要求：

1. select 和输入有 label。
2. 周期按钮有 `aria-pressed`。
3. 趋势 Tab 有 tab 语义。
4. dialog 有 `role="dialog"` 和 `aria-modal`。
5. Escape 可关闭抽屉。
6. 关闭后焦点返回。
7. disabled 操作有说明。
8. 状态不只依赖颜色。
