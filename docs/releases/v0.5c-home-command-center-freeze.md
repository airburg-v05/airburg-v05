# V0.5C Home Command Center Freeze

完成时间：2026-06-22

状态：PASS，待 immutable completion record 注册后冻结。

## 阶段名称

V0.5C：首页经营命令中心。

## 首页定位

V0.5C 将 `/home` 从长模块堆叠页收口为多平台、多店铺经营命令中心。首页只回答：

1. 当前整体经营表现如何。
2. 哪些店铺贡献最高或需要优先查看。
3. 当前日期范围的数据是否完整。
4. 现有目标完成情况如何。
5. 下一步应进入哪个店铺或数据页面。

## V2-first

首页优先读取 active V2 dataset。V2 有效时，首页展示跨店铺汇总、日期周期、店铺筛选、主趋势图和店铺表现入口。

页面不得直接打开 IndexedDB、不得直接读取 object store、不得在加载时自动迁移或写入 V2。

## legacy fallback

当 V2 为空但旧版天猫单店分析结果有效时，首页显示 legacy fallback：

1. 仅展示天猫默认店铺。
2. 不自动迁移。
3. 不创建 active dataset。
4. 不修改 legacy key。
5. 显示轻量提示：当前显示旧版单店数据，完成新数据导入后可查看多店铺汇总。

当 V2 corrupted 且 legacy 有效时，首页仍可 fallback，并显示轻量风险提示。

## 多店铺汇总

V0.5C 支持 active V2 中的实际平台和实际店铺：

1. 全部平台。
2. 实际有数据的平台。
3. 全部店铺。
4. 实际有数据的店铺。

同一 productId 跨店铺不合并。同一 planId 跨店铺不合并。店铺表现按当前范围 GMV 降序展示。

## 日期周期

首页支持：

1. 日。
2. 周，自然周，周一至周日。
3. 月，自然月。
4. 自定义范围，起止日期均包含。

只汇总实际存在的经营日期，不把缺失日期补为 0，不用系统当前日期替代缺失 businessDate。

## 六个核心指标

首页核心指标最多六张卡：

1. GMV。
2. GSV。
3. 商品访客。
4. 支付买家。
5. 支付转化率。
6. 推广花费与 ROI。

## 指标口径

1. GMV：当前范围安全求和。
2. GSV：当前范围安全求和。
3. 商品访客：当前范围安全求和。
4. 支付买家：当前范围安全求和。
5. 支付转化率：总支付买家 / 总访客，不平均商品或店铺转化率。
6. 推广花费与 ROI：遵守冻结推广口径，不重复累计商品推广和计划推广。
7. 缺失来源显示 `--`。
8. 真实数值 0 保留为 0。
9. null 不转换为 0。

默认天猫店铺同一真实样表下，V2 首页核心指标必须与 legacy 冻结口径对账。

## 目标只读规则

V0.5C 首页只读取已有目标，不创建、不编辑、不删除目标。

1. 日目标只用于日视图。
2. 月目标只用于月视图。
3. 周和自定义范围没有明确目标时，不自动折算。
4. 不把月目标平均后伪装成周目标。
5. 不自动生成公司目标。
6. 不把店铺、商品或系列目标合计成公司目标。
7. 无匹配目标时显示当前周期暂无目标。
8. 完成率继续遵守 V0.4C 冻结口径。

## 单一主趋势图

首页只保留一个主趋势图，通过指标 Tab 切换：

1. GMV。
2. GSV。
3. 访客。
4. 支付买家。
5. 转化率。
6. 推广花费。

单日期只展示当日值，不解释为上涨或下降。缺失日期保留缺口，不补 0。

## 店铺表现

店铺表现区展示当前范围内的实际店铺：

1. 平台。
2. 店铺名称。
3. GMV。
4. GSV。
5. GMV 贡献占比。
6. 支付转化率。
7. 推广花费。
8. ROI。
9. 目标完成率，如存在安全匹配目标。
10. 店铺入口。

## 默认店铺下钻

天猫默认店铺允许进入旧店铺看板：

`/store-board?platform=tmall&storeId=tmall-default-store`

## 第二店铺安全限制

V0.5D 多店铺店铺看板尚未实施前，非默认店铺不得进入旧 `/store-board`。

非默认店铺展示：

1. 禁用按钮：店铺看板待升级。
2. 可用入口：查看导入记录，跳转 `/upload/history` 并携带 platform、storeId、batchId。

这可以防止第二店铺数据被旧单店看板误显示成默认店铺数据。

## 轻量数据状态

首页只保留轻量数据状态：

1. 数据状态。
2. 平台数。
3. 店铺数。
4. 经营日期覆盖。
5. safe warning 数量。
6. 数据质量入口。

不在首页展开完整数据质量中心。

## empty / corrupted

1. empty：提示当前没有经营数据，主操作为前往数据导入。
2. corrupted：提示本地经营数据不可安全读取，提供数据质量入口。
3. 不展示损坏对象。
4. 不展示技术堆栈。

## 移动端结果

390px 移动端验收通过：

1. 无整页横向溢出。
2. 周期、平台、店铺筛选可操作。
3. 六个核心指标安全排列。
4. 主趋势图不撑破页面。
5. 店铺表现区域不撑破页面。
6. 第二店铺禁用状态可读。

## 可访问性

1. 周期按钮使用 `aria-pressed`。
2. 指标 Tab 使用 tab 语义和当前选中状态。
3. 平台、店铺、日期输入有 label。
4. 按钮声明 type。
5. 图表有文本摘要。
6. disabled 店铺按钮有说明。
7. 状态不只依赖颜色表达。

## 隐私边界

首页不得输出：

1. 售后原始明细。
2. rawRows。
3. previewRows。
4. 文件名历史。
5. 原始 warning。
6. 订单编号。
7. 退款编号。
8. 交易号。
9. 联系方式。
10. 地址。
11. 物流信息。
12. 买家说明。
13. 商家备注。
14. 操作人或子账号。

售后只允许以安全聚合指标进入首页。
