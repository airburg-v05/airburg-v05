# V0.5D Multi-store Store Board Freeze

冻结时间：2026-06-22T20:20:00+08:00

阶段名称：V0.5D：多店铺 Store Board 基础版

状态：PASS，待 immutable completion record 注册后作为 V0.5D 冻结基线。

## 1. Store Board 定位

V0.5D 将 `/store-board` 从旧版单天猫店铺看板升级为多平台、多店铺上下文下的店铺经营看板。当前重点是让运营安全查看一个具体店铺在指定经营范围内的表现，并确认店铺切换、URL 上下文、目标只读、趋势和运营关注模块不会发生跨店铺污染。

当前版本不重做 Product Board、Series Board、Targets、Upload、Raw Data，也不接入 AI、后端接口、服务器数据库、平台 API 或爬虫。

## 2. V2-first 与 legacy fallback

Store Board 读取顺序保持：

1. 优先读取 active V2 dataset。
2. V2 有效时使用 platform/store 上下文渲染多店铺店铺数据。
3. V2 为空但 legacy 有效时，仅允许天猫默认店铺进入 legacy fallback。
4. 非默认店铺不得使用旧版 legacy fallback。
5. V2 corrupted 时展示安全修复状态，不展示损坏对象。
6. empty/corrupted 状态均不展示技术堆栈。

页面不得自动迁移、不得写入 V2、不得修改 legacy key。

## 3. platform/store 上下文

Store Board 必须携带并识别：

1. `platform`
2. `storeId`

页面顶部持续展示当前店铺上下文。店铺切换会同步 URL，刷新和浏览器前进/后退后仍保持当前店铺。长店铺名必须安全截断，ID 只作弱化信息，不作为主要展示。

## 4. 多店铺隔离

冻结规则：

1. 默认店铺与第二店铺指标隔离。
2. 同 productId 跨店铺不得合并。
3. 同 planId 跨店铺不得合并。
4. 售后安全聚合按 storeId 隔离。
5. 目标按 store scope 隔离。
6. 系列按 storeId 隔离。
7. 商品 TOP 仅来自当前店铺。
8. 系列进度仅来自当前店铺。

## 5. 日期周期

支持：

1. 日。
2. 周。
3. 月。
4. 自定义范围。

日期规则：

1. 只汇总实际存在的经营日期。
2. 缺失日期不补 0。
3. 自定义 start > end 时显示安全错误，不执行计算。
4. 单日期只展示单点数据，不解释为上涨或下降。

## 6. 六个核心指标

核心指标保持最多 6 张卡：

1. GMV。
2. GSV。
3. 商品访客。
4. 支付买家。
5. 支付转化率。
6. 推广花费与 ROI。

口径冻结：

1. GMV/GSV/访客/买家安全求和。
2. 支付转化率 = 总支付买家 / 总商品访客。
3. 推广不重复计算。
4. 计划推广不补商品推广。
5. 缺失推广显示 `--`。
6. 真实 0 保留为 0。
7. null 不转 0。

## 7. 店铺目标只读

Store Board 只读展示当前店铺目标：

1. 不创建目标。
2. 不编辑目标。
3. 不展示目标编辑表单。
4. 日目标仅用于日视图。
5. 月目标仅用于月视图。
6. 周和自定义不自动折算目标。
7. 最多展示 4 个目标。
8. 无目标时显示简洁空状态。

目标设置入口继续跳转 `/targets`。

## 8. 单一主趋势图

Store Board 只保留一个单一主趋势图，通过 Tab 切换：

1. GMV。
2. GSV。
3. 访客。
4. 支付买家。
5. 转化率。
6. 推广花费。

图表必须有文本摘要。不得恢复多个趋势卡片墙，不得输出 NaN、Infinity 或 undefined。

## 9. 运营关注 Tabs

运营关注区使用 Tabs：

1. 商品 TOP：最多 5 条，仅当前店铺。
2. 系列进度：最多 5 条，仅当前店铺，无系列时显示安全空状态。
3. 推广与售后：推广缺失显示 `--`，售后只展示安全聚合。

不得自动 tracked 全部商品，不得自动创建系列，不得展示售后明细或原始原因文本。

## 10. 首页下钻

首页店铺表现区所有实际店铺均可进入 `/store-board`，并携带安全 platform/store context。

默认天猫店铺：

1. 可从首页进入 Store Board。
2. Store Board 内可进入旧 Product Board。
3. Store Board 内可进入旧 Series Board。

非默认商品/系列旧看板限制：

非默认店铺：

1. 可从首页进入 Store Board。
2. 不得进入旧 Product Board。
3. 不得进入旧 Series Board。
4. 商品和系列旧看板按钮显示待升级并 disabled。
5. 可进入对应导入记录和数据质量页面。

## 11. invalid store、empty/corrupted

安全状态：

1. invalid platform：显示当前店铺不可用，不 fallback 到默认店铺。
2. invalid store：显示当前店铺不可用，不 fallback 到默认店铺。
3. empty：提示前往数据导入。
4. corrupted：提示查看数据质量或重新导入。
5. non-default legacy fallback：禁止。

不得展示损坏对象、技术堆栈或跨店铺残留数据。

## 12. 移动端和可访问性

390px 移动端必须满足：

1. 无整页横向溢出。
2. 店铺切换可操作。
3. 周期控件安全换行或局部滚动。
4. 六指标单列或两列。
5. 目标摘要可读。
6. 主趋势图不撑破页面。
7. 运营 Tabs 局部滚动。
8. 商品和系列列表不撑破。
9. 长店铺名不破坏布局。

可访问性要求：

1. 平台和店铺 select 有 label。
2. 周期按钮有 `aria-pressed`。
3. 趋势 Tab 和运营关注 Tab 有 tab 语义。
4. disabled 下钻有说明。
5. 图表有文本摘要。
6. 状态不只依赖颜色。

## 13. 隐私边界

售后只展示安全聚合，不得输出：

1. rawRows。
2. previewRows。
3. 文件名历史。
4. warning 原文。
5. 订单、退款、交易标识。
6. 电话、地址、物流。
7. 买家说明、商家备注、操作人。

最终回归确认敏感值泄漏为 0。
