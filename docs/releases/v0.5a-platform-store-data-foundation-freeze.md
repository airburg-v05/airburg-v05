# V0.5A 平台店铺数据地基冻结说明

## 1. 阶段名称

V0.5A：平台、店铺与数据归属模型。

冻结任务：

`V0.5A_5_R2_FINAL_REGRESSION_AND_STAGE_FREEZE`

冻结时间：

2026-06-22

## 2. 阶段定位

V0.5A 完成的是多平台、多店铺的数据归属与本地持久化地基。

本阶段不重做当前页面，不改变 V0.4E 页面行为。当前页面仍使用 legacy 数据链路，V2 Repository 和 IndexedDB persistence 仅作为后续 V0.5B 批量导入的地基能力。

## 3. 已完成范围

V0.5A 已完成：

1. 多平台、多店铺归属模型。
2. `platformCode` / `storeId` / `businessDate` / `sourceType` / `importBatchId` 所有权字段。
3. 默认天猫店铺归属策略。
4. Storage V2 domain model。
5. V2 repository contract。
6. Memory repository adapter。
7. V2 record validators。
8. 旧 localStorage 五 key snapshot 读取。
9. 旧单天猫数据 dry-run。
10. 售后安全 daily / range / operational snapshot / distribution 聚合类型。
11. 真实四源样表 readiness gate。
12. IndexedDB prepare / readback / activate。
13. already_active / second activation。
14. rollback / recovery。
15. 真实 Chrome 生产 Adapter 集成验证。
16. 阶段冻结治理验证。

## 4. 默认天猫店铺

旧单天猫数据迁移归属采用默认店铺：

1. `platformCode = tmall`
2. `storeId = tmall-default-store`

旧数据不得通过清空、覆盖或伪造方式绕开迁移。

## 5. Legacy 数据链路

当前页面仍读取旧链路，不自动迁移到 V2：

1. 首页仍读取 legacy 天猫分析结果。
2. 上传页仍保留原四源上传结果逻辑。
3. 店铺、系列、宝贝、目标页面仍使用现有 legacy storage。
4. 页面没有自动创建 production IndexedDB。
5. 页面没有自动执行 migration。
6. 页面没有切换 active dataset。

## 6. 五个旧 Key 全部保留

V0.5A 保留并不自动修改以下旧 key：

1. `airburg_tmall_analysis_v2`
2. `airburg_tmall_series_groups_v1`
3. `airburg_tmall_targets_v1`
4. `airburg:last-analysis`
5. `airburg:demo-session`

其中：

1. `airburg_tmall_analysis_v2` 仍是当前页面读取的天猫分析结果。
2. `airburg_tmall_series_groups_v1` 仍是当前系列分组存储。
3. `airburg_tmall_targets_v1` 仍是当前目标存储。
4. `airburg:last-analysis` 是旧预览分析产物，不进入 V2 business repository。
5. `airburg:demo-session` 是演示登录状态，不进入 V2 business repository。

V0.5A 不自动删除、覆盖或清空任何 legacy key。

## 7. Dry-run

Dry-run 已完成：

1. 读取完整五 key snapshot。
2. 基于 analysis / series / targets 业务 payload 生成 business dataset fingerprint。
3. 基于完整五 key snapshot 生成 manifest fingerprint。
4. 只 staging `parsed` 来源事实。
5. source health 与事实数组不一致时输出安全 issue。
6. 保留合法 ad-only product fact。
7. 不写 localStorage。
8. 不写 IndexedDB。
9. 不写 active pointer。

历史 A-2、A-3、A-3.1、A-3.2 的 task-scoped 脚本不在当前冻结任务中重新执行；它们的 PASS 证据来自不可变 completion records。

## 8. IndexedDB 持久化

V0.5A 已实现本地 IndexedDB V2 persistence foundation。

已验证：

1. `prepare = prepared`
2. `readback = readback_validated`
3. `activate = activated`
4. repeated activation = `already_active`
5. second dataset activation = `activated`
6. `rollback = rolled_back`
7. failure injection 四项通过
8. pointer 原子性通过
9. record counts / keys / journal 对账通过
10. audit DB 已清理
11. 未触碰 production DB

13. 子账号。
14. 物流单号。
15. 物流信息。
