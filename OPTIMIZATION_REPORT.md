## 4. 底层修复

### 4.1 修复空表头导致列错位

原解析器会删除空表头后再按新数组索引读取数据，可能造成后续列整体左移。本轮改为保留原列索引，只忽略空表头对应列。

### 4.2 修复重复表头覆盖

重复表头会增加唯一后缀，避免对象字段直接覆盖；字段映射会把重复候选标记为待确认。

### 4.3 修复同名商品错误合并

商品排行和异常识别现在优先按商品 ID 聚合，商品 ID 缺失时才回退到商品名称。

### 4.4 过滤合计行

当文件同时存在正常商品行和“合计 / 总计 / 汇总”等行时，自动移除合计行，避免总指标重复计算。

### 4.5 改进字段质量

字段质量从简单的“有效 / 缺失 / 无效”扩展为：

- 完整。
- 部分缺失。
- 未识别。
- 格式异常。

并保留有效、缺失和异常数量。

### 4.6 增加文件保护

- 只接受 CSV / XLS / XLSX。
- 单文件最大 25MB。
- 最多 10 万行。
- 不支持格式时给出明确错误，不再返回空结果。

### 4.7 改进 CSV 编码兼容

优先按 UTF-8 读取；UTF-8 解码失败时尝试 GB18030，降低中文导出 CSV 乱码概率。

### 4.8 修复总转化率为 0 的情况

有访客但支付买家为 0 时，转化率现在正确返回 `0%`，不再显示为空。

## 8. 已知限制与风险

### 8.1 SheetJS `xlsx` 安全公告

当前 `xlsx@0.18.5` 存在公开安全公告，并且 npm 当前没有直接修复版本。项目仍需支持旧版 `.xls`，因此本轮保留该依赖，同时增加文件大小和行数限制。

后续正式部署前应重新评估 Excel 解析方案，不应允许不可信用户无限制上传文件。

### 8.2 Next.js 间接 PostCSS 公告

`npm audit` 仍会报告 Next.js 间接依赖中的 PostCSS 中等级别问题。不要执行 `npm audit fix --force`，因为它会提出不合理的破坏性降级。

### 8.3 当前不是正式认证

登录页仅用于建立产品流程和页面结构，不能当作真实安全登录。

### 8.4 当前无数据库

刷新后页面本身不会保留 React 状态，但最近一次结果会保存在当前浏览器 localStorage；换浏览器、清理缓存或更换电脑后会消失。

### 8.5 多文件合并尚未完成

生意参谋商品表和推广商品报表目前可以分别解析，尚未按商品 ID 和日期合并。因此商品级 18 项完整指标、趋势和目标尚未全部开放。

## 9. 推荐下一开发任务

下一任务不要继续铺更多静态页面，应该优先完成：

**天猫双数据源关联与数据预检。**

具体包括：

1. 同时选择一份生意参谋商品表和一份推广商品报表。
2. 显示两份文件各自字段识别结果。
3. 让用户确认商品 ID / 主体 ID 和日期字段。
4. 按商品 ID + 日期合并。
5. 输出匹配成功、未匹配商品和重复数据统计。
6. 为宝贝看板提供完整 18 项指标的数据结构。

完成这一任务后，再开发目标设置和 MTD / DLY 趋势图，进度会更稳。
