# Airburg Data 电商数据分析平台

这是电商数据分析平台的天猫 V1 优化版。当前重点是把“本地上传文件 → 字段识别 → 数据清洗 → 指标计算 → 首页与看板展示”跑通。

## 当前可用页面

| 页面 | 路径 | 当前能力 |
|---|---|---|
| 登录页 | `/login` | 本地演示登录，已预填测试账号 |
| 首页 | `/home` | 自动读取最近一次分析结果，展示核心指标、排行和风险提示 |
| 数据上传 | `/upload` | 解析天猫 Excel / CSV，展示字段映射、数据质量、指标和原始预览 |
| 宝贝看板 | `/product-board` | 展示已识别商品及当前可用的基础指标 |
| 系列看板 | `/series-board` | 正式入口和开发边界说明，暂未伪造业务能力 |
| 店铺看板 | `/store-board` | 汇总最近一次上传文件的店铺基础指标 |
| 原始数据 | `/raw-data` | 查看最近一次分析保存的前 50 行数据 |

## 本地启动

在项目根目录依次运行：

```bash
npm install
npm run dev
```

浏览器打开：

```text
http://localhost:3000
```

登录页已经预填演示账号，直接点击“进入工作台”。

## 代码检查

```bash
npm run lint
npm run build
```

当前两个命令均可通过。

## 当前数据边界

- 文件在浏览器本地解析，不上传服务器。
- 最近一次分析结果保存在浏览器 `localStorage`。
- 支持 `.csv`、`.xls`、`.xlsx`。
- 单文件限制 25MB，最多 10 万行。
- 当前可以独立解析生意参谋商品表或推广商品报表。
- 商品表与推广表的自动合并尚未实现。
- 正式账号、数据库、历史上传、多店铺权限和 AI 报告尚未实现。
